# 校园活动发布平台需求分析

## 1. 项目背景

校园活动发布平台是一个为高校师生提供活动信息发布、浏览、管理的综合性平台。该平台旨在简化活动发布流程，提高活动信息的传播效率，同时确保活动内容的质量和合规性。

## 2. 核心功能需求

### 2.1 用户管理
- **注册功能**：支持手机号和邮箱注册
- **登录功能**：支持账号密码登录
- **个人中心**：用户信息管理、活动管理

### 2.2 活动管理
- **活动发布**：用户可以发布校园活动，包括活动名称、时间、地点、描述、海报等信息
- **活动编辑**：发布者可以编辑已发布的活动信息
- **活动删除**：发布者可以删除已发布的活动
- **活动列表**：用户可以浏览所有活动信息
- **活动详情**：用户可以查看活动的详细信息

### 2.3 审核系统
- **人工审核**：审核员可以审核活动内容
- **AI辅助审核**：使用AI技术辅助审核活动内容
- **审核开关**：管理员可以控制是否启用AI辅助审核功能
- **审核状态**：活动有待审核、审核通过、审核拒绝等状态

### 2.4 搜索与筛选
- **活动搜索**：用户可以通过关键词搜索活动
- **活动筛选**：用户可以按时间、类型、地点等条件筛选活动

### 2.5 通知系统
- **活动状态通知**：活动审核状态变更通知
- **活动提醒**：活动开始前的提醒

## 3. 技术需求

### 3.1 前端技术栈
- **框架**：Vue 3
- **语言**：TypeScript + JavaScript
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI组件库**：Element Plus
- **HTTP请求**：Axios

### 3.2 后端技术栈
- **语言**：Java
- **框架**：Spring Boot
- **数据库**：MySQL
- **缓存**：Redis
- **认证**：JWT
- **AI集成**：集成AI模型进行内容审核

### 3.3 其他技术
- **文件存储**：本地存储或云存储
- **部署**：Docker

## 4. 非功能需求

### 4.1 性能需求
- 页面加载时间不超过3秒
- 活动列表查询响应时间不超过1秒

### 4.2 安全需求
- 密码加密存储
- JWT token认证
- 防止SQL注入
- 防止XSS攻击

### 4.3 可用性需求
- 系统可用性达到99.9%
- 支持断点续传

### 4.4 可扩展性需求
- 模块化设计
- 支持水平扩展

## 5. 模拟功能

- **支付功能**：由于无法使用需要注册资格的功能，支付功能将作为模拟功能实现，不进行实际支付操作

## 6. 免费注册渠道

- **手机号注册**：使用短信验证码验证
- **邮箱注册**：使用邮箱验证码验证

## 7. AI辅助审核

- **功能描述**：使用AI模型对活动内容进行自动审核，检测违规内容
- **开关控制**：管理员可以在后台控制是否启用AI辅助审核功能
- **审核流程**：AI审核后，人工审核员可以查看AI审核结果并进行最终审核

## 8. 项目范围

- 本项目仅包含校园活动发布平台的核心功能
- 不包含复杂的社交功能
- 不包含实际的支付功能
- 重点关注活动发布、审核和浏览功能
