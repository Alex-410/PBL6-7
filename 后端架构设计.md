# 校园活动发布平台后端架构设计

## 1. 技术栈

- **语言**：Java 17.0.12
- **框架**：Spring Boot 3.2.4
- **数据库**：MySQL 8.0.44
- **认证**：JWT
- **ORM**：MyBatis 3.0.3
- **API文档**：Swagger/OpenAPI (待实现)
- **构建工具**：Maven 3.9.11
- **安全框架**：Spring Security

## 3. 核心功能模块

### 3.1 认证模块
- **用户注册**：手机号注册、邮箱注册
- **用户登录**：账号密码登录
- **密码重置**：通过手机号或邮箱重置密码
- **Token管理**：JWT token生成、验证、刷新

### 3.2 活动模块
- **活动发布**：创建新活动
- **活动编辑**：编辑已发布的活动
- **活动删除**：删除已发布的活动
- **活动查询**：获取活动列表、活动详情
- **活动状态管理**：活动状态变更

### 3.3 审核模块
- **人工审核**：审核员审核活动内容
- **AI辅助审核**：使用AI技术辅助审核活动内容
- **审核设置**：管理员控制AI辅助审核开关
- **审核记录**：审核操作记录

### 3.4 用户模块
- **用户信息管理**：获取、更新用户信息
- **用户角色管理**：用户角色分配
- **用户权限管理**：权限控制

### 3.5 通知模块
- **通知发送**：发送系统通知
- **通知管理**：获取、标记、删除通知

## 7. AI辅助审核设计

### 7.1 AI审核流程
1. 活动发布后，检查是否启用AI审核
2. 如果启用，调用AI模型对活动内容进行审核
3. AI审核完成后，将审核结果存储到数据库
4. 人工审核员可以查看AI审核结果并进行最终审核

### 7.2 AI模型集成
- **模型选择**：使用预训练的文本分类模型
- **审核内容**：活动标题、描述、组织者等文本内容
- **审核结果**：正常、疑似违规、违规
- **阈值设置**：可配置的审核阈值

### 7.3 审核开关控制
- 管理员可以在后台控制是否启用AI辅助审核功能
- 可以设置AI审核的阈值
