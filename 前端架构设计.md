# 校园活动发布平台前端架构设计

## 1. 技术栈

- **框架**：Vue 3
- **语言**：TypeScript + JavaScript
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI组件库**：Element Plus
- **HTTP请求**：Axios
- **构建工具**：Vite
- **CSS预处理器**：SCSS

## 2. 项目结构

```
src/
├── assets/            # 静态资源
│   ├── images/        # 图片资源
│   └── styles/        # 全局样式
├── components/        # 通用组件
│   ├── layout/        # 布局组件
│   ├── common/        # 通用组件
│   └── activity/      # 活动相关组件
├── views/             # 页面组件

│   ├── activity/      # 活动相关页面

```

## 3. 核心功能模块

### 3.1 认证模块
- **注册功能**：手机号注册、邮箱注册
- **登录功能**：账号密码登录
- **找回密码**：通过手机号或邮箱找回密码

### 3.2 活动模块
- **活动发布**：创建新活动
- **活动编辑**：编辑已发布的活动
- **活动删除**：删除已发布的活动
- **活动列表**：浏览活动列表
- **活动详情**：查看活动详情

### 3.3 审核模块
- **审核列表**：审核员查看待审核的活动
- **审核操作**：通过或拒绝活动
- **AI审核设置**：管理员控制AI辅助审核开关

### 3.4 搜索与筛选模块
- **关键词搜索**：通过关键词搜索活动
- **条件筛选**：按时间、类型、地点等筛选活动

### 3.5 通知模块
- **消息列表**：查看系统通知
- **通知设置**：设置通知偏好

## 6. 路由设计

```typescript
const routes = [
  // 公共路由
  {
    path: '/',
    component: HomePage
  },
  {
    path: '/login',
    component: LoginPage
  },
  {
    path: '/register',
    component: RegisterPage
  },
  {
    path: '/activity/:id',
    component: ActivityDetailPage
  },
  {
    path: '/activities',
    component: ActivityListPage
  },
  
  // 需要认证的路由
  {
    path: '/user',
    component: UserLayout,
    meta: { requiresAuth: true },
    children: [
      {
        path: 'profile',
        component: UserProfilePage
      },
      {
        path: 'activities',
        component: UserActivityPage
      },
      {
        path: 'notifications',
        component: NotificationPage
      }
    ]
  },
  
  // 审核员路由
  {
    path: '/audit',
    component: AuditLayout,
    meta: { requiresAuth: true, requiresAuditor: true },
    children: [
      {
        path: 'list',
        component: AuditListPage
      },
      {
        path: 'detail/:id',
        component: AuditDetailPage
      }
    ]
  },
  
  // 管理员路由
  {
    path: '/admin',
    component: AdminLayout,
    meta: { requiresAuth: true, requiresAdmin: true },
    children: [
      {
        path: 'users',
        component: UserManagementPage
      },
      {
        path: 'settings',
        component: SettingsPage
      },
      {
        path: 'ai-settings',
        component: AISettingsPage
      }
    ]
  }
];
```

## 7. API接口设计

### 7.1 认证接口
- **POST /api/auth/register**：注册
- **POST /api/auth/login**：登录
- **POST /api/auth/reset-password**：重置密码
- **POST /api/auth/refresh-token**：刷新token

### 7.2 活动接口
- **GET /api/activities**：获取活动列表
- **GET /api/activities/:id**：获取活动详情
- **POST /api/activities**：创建活动
- **PUT /api/activities/:id**：更新活动
- **DELETE /api/activities/:id**：删除活动

### 7.3 审核接口
- **GET /api/audit/activities**：获取待审核活动列表
- **PUT /api/audit/activities/:id**：审核活动
- **GET /api/audit/settings**：获取审核设置
- **PUT /api/audit/settings**：更新审核设置

### 7.4 用户接口
- **GET /api/users/profile**：获取用户信息
- **PUT /api/users/profile**：更新用户信息
- **GET /api/users/activities**：获取用户发布的活动

### 7.5 通知接口
- **GET /api/notifications**：获取通知列表
- **PUT /api/notifications/:id/read**：标记通知为已读
- **DELETE /api/notifications/:id**：删除通知
