# 校园活动发布平台需求文档 (PRD)

## 📌 需求概述

- **需求背景**: 当前校园活动信息分散，学生难以获取全面的活动资讯；同时学生在筛选可加分活动、管理综合测评分数时缺乏便捷工具，导致活动参与率低、加分管理混乱。
- **业务价值**: 打造一站式校园活动发布与管理平台，解决活动信息分散、加分筛选困难的问题，提升学生活动参与体验，规范活动加分流程，提高校园活动运营效率。

---

## 👥 用户角色

| 角色名称 | 权限/特征描述 |
|---|---|
| 学生 | 浏览活动、报名参加活动、取消报名、查看个人加分记录、提交加分申诉、发布小型活动 |
| 社团负责人 | 发布社团活动、管理活动报名名单、上传参与学生名单、申请活动加分 |
| 学院管理员 | 发布学院活动（可设置参与学院限制）、管理活动报名名单、上传参与学生名单、申请活动加分 |
| 学校管理员 | 发布校级活动、审核所有活动上架申请、确认加分名单、处理加分申诉、管理平台用户 |
| 校外商家 | 发布商业活动（需审核）、无加分权限 |

---

## 🔄 业务流程

### 1. 活动发布流程

1. 发布者（社团负责人/学院管理员/学校管理员/校外商家/学生）登录平台
2. 填写活动信息（标题、类型、时间、地点、人数限制、参与身份限制、是否可加分、加分类型及数量等）
3. 提交活动申请
4. 平台管理员审核活动
5. 审核通过 → 活动上架展示；审核拒绝 → 返回拒绝原因，发布者可修改后重新提交

### 2. 学生浏览与报名流程

1. 学生登录平台
2. 浏览活动列表（可按类型、时间、是否可加分、学院等筛选）
3. 查看活动详情
4. 点击报名
5. 系统校验学生身份是否符合活动参与限制
6. 校验通过 → 报名成功，收到确认通知；校验不通过 → 提示不符合报名条件

### 3. 取消报名流程

1. 学生进入"我的报名"页面
2. 选择已报名的活动
3. 点击取消报名
4. 系统校验是否在活动开始前24小时内
5. 校验通过 → 取消成功，释放名额；校验不通过 → 提示"活动开始前24小时内不可取消"

### 4. 活动加分流程

1. 活动结束后，发布者上传参与学生名单
2. 学校管理员/平台管理员确认名单
3. 系统自动为学生添加对应活动分数（仅限学校管理员/学院/社团发布的活动）
4. 学生可在个人中心查看加分记录

### 5. 加分申诉流程

1. 学生发现加分有误，进入加分记录页面
2. 选择有异议的记录，提交申诉（附上证明材料）
3. 学校管理员审核申诉
4. 审核通过 → 更新加分；审核拒绝 → 返回拒绝原因

## 📝 功能需求明细

### 模块一：用户管理

#### 1.1 用户注册与登录

**作为** 学生/发布者，**我希望** 能够使用学校统一身份认证登录平台，**以便于** 快速访问系统并确保身份真实。

- **验收标准 (AC)**:
  - Given 用户已拥有学校统一身份账号 When 用户使用统一身份认证登录 Then 系统自动完成注册/登录并跳转至首页
  - Given 用户登录失败 When 账号或密码错误 Then 系统提示"账号或密码错误"

#### 1.2 用户信息管理

**作为** 用户，**我希望** 能够查看和补充个人基本信息，**以便于** 系统准确识别我的身份（如所属学院）。

- **验收标准 (AC)**:
  - Given 用户已登录 When 用户进入个人中心 Then 可查看学号、姓名、学院、年级等基本信息
  - Given 用户信息不完整 When 用户补充信息 Then 系统保存并更新用户档案

---

### 模块二：活动管理

#### 2.1 活动发布

**作为** 发布者，**我希望** 能够发布活动并填写详细信息，**以便于** 向目标用户宣传活动。

- **验收标准 (AC)**:
  - Given 发布者已登录 When 填写活动信息（标题、类型、时间、地点、人数限制、参与身份限制、加分信息）并提交 Then 系统创建活动申请并进入待审核状态
  - Given 活动需要加分 When 发布者为学校管理员/学院/社团 Then 可设置加分类型及数量
  - Given 发布者为学生 When 发布活动 Then 系统标记为"学生活动"，无加分权限

#### 2.2 活动审核

**作为** 平台管理员，**我希望** 能够审核活动申请，**以便于** 确保活动内容合规。

- **验收标准 (AC)**:
  - Given 管理员进入审核列表 When 查看待审核活动 Then 可查看活动详情并选择通过或拒绝
  - Given 管理员拒绝活动 When 填写拒绝原因 Then 系统通知发布者并返回拒绝原因
  - Given 发布者收到拒绝通知 When 修改活动信息并重新提交 Then 活动重新进入待审核状态

#### 2.3 活动列表展示

**作为** 学生，**我希望** 能够浏览活动列表并进行筛选，**以便于** 快速找到感兴趣的活动。

- **验收标准 (AC)**:
  - Given 学生进入活动列表页 When 页面加载完成 Then 展示所有已上架活动（按时间倒序）
  - Given 学生使用筛选功能 When 选择筛选条件（类型/是否可加分/学院/时间范围） Then 列表实时更新展示符合条件的活动
  - Given 活动已报满 When 学生查看活动列表 Then 活动显示"已报满"标识

#### 2.4 活动详情展示

**作为** 学生，**我希望** 能够查看活动详细信息，**以便于** 决定是否报名。

- **验收标准 (AC)**:
  - Given 学生点击活动卡片 When 进入活动详情页 Then 展示活动标题、时间、地点、主办方、参与限制、加分信息、剩余名额等
  - Given 活动有参与身份限制 When 学生不符合条件 Then 报名按钮置灰并提示"该活动仅限XX学院学生参加"

---

### 模块三：报名管理

#### 3.1 活动报名

**作为** 学生，**我希望** 能够报名参加活动，**以便于** 参与感兴趣的校园活动。

- **验收标准 (AC)**:
  - Given 活动未报满且学生符合参与条件 When 学生点击报名 Then 报名成功，名额减一，学生收到确认通知
  - Given 活动已报满 When 学生点击报名 Then 系统提示"活动已报满"
  - Given 学生不符合参与身份限制 When 学生点击报名 Then 系统提示"您不符合该活动的参与条件"

#### 3.2 取消报名

**作为** 学生，**我希望** 能够取消已报名的活动，**以便于** 释放名额给其他同学。

- **验收标准 (AC)**:
  - Given 学生已报名且活动开始时间>24小时 When 学生点击取消报名 Then 取消成功，名额释放，系统通知学生
  - Given 活动开始时间≤24小时 When 学生点击取消报名 Then 系统提示"活动开始前24小时内不可取消报名"

#### 3.3 我的报名

**作为** 学生，**我希望** 能够查看我已报名的所有活动，**以便于** 管理我的活动日程。

- **验收标准 (AC)**:
  - Given 学生进入"我的报名"页面 When 页面加载完成 Then 展示已报名活动列表（按时间排序）
  - Given 活动已结束 When 学生查看列表 Then 活动显示"已结束"状态

---

### 模块四：加分管理

#### 4.1 加分名单上传

**作为** 活动发布者（学校管理员/学院/社团），**我希望** 能够上传参与学生名单，**以便于** 为学生添加活动分数。

- **验收标准 (AC)**:
  - Given 活动已结束 When 发布者上传参与学生名单（Excel/手动输入） Then 系统解析名单并展示预览
  - Given 名单中有非报名学生 When 发布者确认名单 Then 系统提示异常数据供发布者确认

#### 4.2 加分确认

**作为** 学校管理员，**我希望** 能够确认加分名单，**以便于** 确保加分数据准确。

- **验收标准 (AC)**:
  - Given 发布者提交加分名单 When 学校管理员审核确认 Then 系统自动为学生添加对应分数
  - Given 学校管理员发现名单有误 When 修改名单后确认 Then 系统按修改后名单加分

#### 4.3 加分记录查询

**作为** 学生，**我希望** 能够查看我的加分记录，**以便于** 了解综合测评分数情况。

- **验收标准 (AC)**:
  - Given 学生进入"加分记录"页面 When 页面加载完成 Then 展示所有加分记录（活动名称、加分类型、加分数量、加分时间）
  - Given 学生按加分类型筛选 When 选择某一项学分 Then 列表展示该项学分的所有加分记录

#### 4.4 加分申诉

**作为** 学生，**我希望** 能够对加分记录提出申诉，**以便于** 修正错误的加分数据。

- **验收标准 (AC)**:
  - Given 学生发现加分有误 When 提交申诉（选择记录、填写原因、上传证明材料） Then 系统创建申诉工单并通知管理员
  - Given 管理员审核申诉通过 When 更新加分数据 Then 系统通知学生申诉结果
  - Given 管理员审核申诉拒绝 When 填写拒绝原因 Then 系统通知学生申诉结果及原因

---

### 模块五：通知中心

#### 5.1 系统通知

**作为** 用户，**我希望** 能够收到系统通知，**以便于** 及时了解活动状态变化。

- **验收标准 (AC)**:
  - Given 学生报名成功 When 系统处理完成 Then 学生收到报名成功通知
  - Given 活动即将开始（提前1天/1小时） When 系统定时任务执行 Then 学生收到活动提醒通知
  - Given 发布者活动审核完成 When 审核结果产生 Then 发布者收到审核结果通知
  - Given 学生加分申诉有结果 When 管理员处理完成 Then 学生收到申诉结果通知

## ⚠️ 非功能需求

### 性能要求
- 活动列表页加载时间 < 2秒
- 报名操作响应时间 < 1秒
- 支持并发报名场景，避免超卖（同一活动同一时间只能被一人报名）

### 安全要求
- 用户登录需通过学校统一身份认证
- 不同角色权限隔离，防止越权操作
- 敏感数据（如学生信息）需脱敏展示

### 兼容性
- 支持 PC 端主流浏览器（Chrome、Edge、Firefox）
- 支持移动端 H5 适配

### 可扩展性
- 加分类型支持后续扩展（当前4项学分，未来可能增加）
- 活动类型支持后续扩展

## 🚫 范围边界

### 包含 (In Scope)
- 用户注册登录（对接学校统一身份认证）
- 活动发布、审核、展示
- 活动报名、取消报名
- 加分名单上传、确认、记录查询
- 加分申诉
- 系统通知
- 本地数据库模拟教务系统（加分数据存储）

### 不包含 (Out of Scope)
- 与学校官方教务系统真实对接
- 活动费用支付功能
- 活动签到功能（如二维码签到）
- 活动评价功能
- 数据统计与报表功能

## ❓ 待确认问题

1. **统一身份认证对接方式**: 学校统一身份认证系统是否提供标准接口（如 OAuth2.0 / CAS）？对接文档是否可获取？
2. **加分类型具体名称**: 学生4项学分的具体名称是什么？（如：德育分、智育分、体育分、美育分）
3. **校外商家活动限制**: 校外商家发布的活动是否需要额外限制（如活动数量上限、活动类型限制）？
4. **活动图片/附件**: 活动发布时是否需要支持上传活动海报图片或附件？
5. **数据统计需求**: 是否需要管理员端的数据统计看板（如活动数量、报名人数、加分统计等）？
